The FAT file system

The FAT file system implementation in eLua uses Elm Chan's excellent FatFS package, available here. It can handle FAT12, FAT16 and FAT32 file systems in read/write mode, and it packs a lot of functionality in a low footprint. eLua adds a platform abstraction layer on top of FatFS which makes it very easy to port the FAT file system implementation between different eLua targets. Since most SD/MMC cards can be accessed using the very common SPI interface, and since eLua already provides a SPI platform interface, porting the FAT file system to an eLua board is a fairly simple process.

To use the FAT file system on your elua board (that has the proper hardware to access SD/MMC cards), first make sure that there is a working implementation of the eLua SPI platform interface on your platform (as currently eLua can access SD/MMC cards only via SPI). Then you need to enable the FAT file system module in your eLua binary image, as decribed in the building page. To open a file on the SD/MMC card, all you need to do is to prefix its name with /mmc/, like this:

# lua /mmc/info.lua
